Buttons

Buttons are interactive components that allow users to take action with a single click or tap.

Usage

Buttons should be placed with associated prompts or related information, easily discoverable on the screen, and not directly adjacent or intersecting another interactive component.

Examples of common button actions:

  • Important calls to action
  • Form submission
  • Affirmation or dismissal of a modal window
  • Navigation within another component

Basic Buttons

There are two main style variants for basic buttons: solid (default) and outline. All basic buttons should include the basic styling:

button {
    border-radius: 4px;
    font-family: ‘Lato’, sans-serif;
    font-size: 16px;
    font-weight: 400;
}

Solid Buttons

Solid buttons have a solid fill and no border. They should be used for primary or affirmative actions.

<button class="solid">Action</button>

Outline Buttons

Outline buttons are transparent or have no fill with a solid border. They should be used for secondary or dismissive actions.

<button class="outline">Action</button>

Button Variations

Basic button styles can be further modified for certain accommodations.

Icon-and-Label Buttons

Button labels can be supported with an added icon to add emphasis or allow users to quickly recognize the button’s intended action.

<button class="solid icon-and-label"><span class="[icon-name]" /> Action</button>

Icon-Only Buttons

Icon-only buttons contain only a single icon in place of the button label. They should be used where space is limited, and the icon should quickly and clearly communicate the proceeding action.

*Accessibility note: Icon-only buttons must have an aria-label to describe the action.

<button class="outline icon-only" aria-label="Action"><span class="[icon-name]" /></button>

Grouped Buttons

Buttons can be grouped if their actions are closely related. All buttons within a group should use the same style variant. The style variant can be indicated in the parent element, and each button within the group will inherit the style of the parent element.

*Accessibility note: Consideration for horizontal spacing should be taken to prevent overflow, as grouped buttons should not wrap and may extend beyond view.

<div class="button-group solid">
    <button>Action</button>
    <button>Action</button>
    <button>Action</button>
</div>

Colors

Buttons may only be implemented in the accepted brand colors along with the extended color palette for alerts.
